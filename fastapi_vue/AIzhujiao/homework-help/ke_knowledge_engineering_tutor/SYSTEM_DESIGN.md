# 系统设计说明

## 1. 系统定位
本系统面向北京科技大学计算机系研究生《知识工程》课程，目标是实现一个可演示、可扩展的课程专用智能助教原型。

## 2. 主要模块

### 2.1 批量查题模块
输入来源包括：
- 多个题目文本
- 多张题目图片
- 单张图片中的多个题目

处理流程：
1. 接收多个文本题目与多张图片。
2. 对文本题目做清洗与编号。
3. 对图片调用模型进行“题目抽取与拆分”。
4. 将全部题目统一成标准题目列表。
5. 逐题调用模型生成结构化解答。
6. 前端以题目列表 + 详情页的形式呈现。

### 2.2 PPT 知识点总结模块
1. 上传 `.pptx`。
2. 使用 `python-pptx` 抽取每页文本。
3. 拼接成结构化讲义内容。
4. 调用模型输出：课程主题、章节结构、核心知识点、每页摘要、易混点、复习建议。
5. 前端支持复制与按块查看。

## 3. Prompt 设计原则
系统 Prompt 重点强调：
- 场景限定为《知识工程》课程
- 默认面向研究生，允许更严谨的术语与方法论表达
- 回答不只给结论，要解释知识表示、推理机制、系统设计思路与工程取舍
- 与大模型结合时，要区分“传统知识工程方法”和“LLM 赋能方法”

## 4. 可扩展方向
- 接入课程知识库与参考教材
- 题目结果持久化与历史记录
- 支持 PDF/PPT 混合解析
- 支持教师端上传作业题单与标准答案
- 增加知识点图谱可视化页
